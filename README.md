CryptoSafe Manager (проект по дисциплине "Методы и средства криптографической защиты информации")

Менеджер паролей с графическим интерфейсом, зашифрованной локальной базой данных и защищенной работой с буфером обмена

### Видение проекта

Идея проекта — сделать простое настольное приложение, в котором:
- пароли и другие данные хранятся в **локальной шифрованной базе SQLite**;
- доступ к хранилищу защищён **мастер‑паролем**;
- действия пользователя (добавление/изменение/удаление) можно в будущем писать в **журнал аудита**;
- есть базовый GUI, который позже можно расширять (буфер обмена, настройки, теги, экспорт и т.д.).

В этом репозитории реализован **Спринт 1** — основа для дальнейших доработок: архитектура, БД,
заглушки шифрования, главное окно, события и тестовая инфраструктура.

### Дорожная карта из 8 спринтов

1. **[Спринт 1](sprints/sprint1.md)** – основа: схема БД, простое шифрование на уровне приложения, главное окно, события, тесты (реализовано здесь).  
2. **[Спринт 2](sprints/sprint2.md)** – управление ключами (`KeyManager`, хранение ключей в таблице `key_store`).  
3. **[Спринт 3](sprints/sprint3.md)** – замена XOR‑заглушки на нормальный AES‑GCM.  
4. **[Спринт 4](sprints/sprint4.md)** – работа с буфером обмена и таймер очистки.  
5. **[Спринт 5](sprints/sprint5.md)** – реальный журнал аудита и подпись записей.  
6. **[Спринт 6](sprints/sprint6.md)** – теги, корзина и дополнительные поля для записей.  
7. **[Спринт 7](sprints/sprint7.md)** – автоблокировка по неактивности пользователя.  
8. **[Спринт 8](sprints/sprint8.md)** – резервное копирование, экспорт/импорт, сборка (Docker/инсталлятор).  

### Как запустить

1. **Создать виртуальное окружение (по желанию)**:
   ```bash
   python -m venv .venv
   .venv\Scripts\activate   # Windows
   ```
2. **Установить зависимости**:
   ```bash
   pip install -r requirements.txt
   ```
3. **Запустить программу**:
   ```bash
   python main.py
   ```
   При первом запуске откроется мастер создания учётной записи (логин + мастер‑пароль)
   и выбора файла базы данных. Потом откроется основное окно с таблицей записей.
4. **Запустить тесты**:
   ```bash
   pytest tests/ -v
   ```

### Структура проекта

- `src/core/` – конфиг, криптография, события, состояние приложения  
- `src/gui/` – PyQt‑интерфейс (окно входа, мастер, главное окно, виджеты)  
- `src/database/` – схема SQLite и помощник для работы с БД  
- `tests/` – модульные и интеграционные тесты  

### Результаты по итогу спринта 1

- **ARC (архитектура)**  
  - Разделение на core/gui/database/tests: см. папки `src/*`, `tests/`.  
  - Менеджер конфигураций: `src/core/config.py` (`Config`, `get_config()`).

  - Заглушка AES‑шифрования на XOR: `src/core/crypto/placeholder.py`.  
  - Заглушка `KeyManager`: `src/core/key_manager.py`.  

- **GUI**  
  - Главное окно (меню, таблица, статус‑бар): `src/gui/main_window.py`.  
  - Окно входа (логин + пароль): `src/gui/login_dialog.py`.

- **Тесты и деплой**
